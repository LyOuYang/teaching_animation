# AP-CSP 基础篇：智能奶茶店长 (Smart Boba Shop Manager)

> **教学目标**：通过运营一家智能奶茶店，让学生在解决实际业务问题的过程中，掌握计算逻辑的基础实现。
> **核心概念**：从配方配置（变量）到计价系统（表达式），再到质检规则（条件）与订单批处理（循环）。
> **视觉风格**：现代清新、高亮数据流、动态液面效果。

## 📚 知识点体系架构

### 🎯 第一阶段：配方配置与存储 (Variables & Types)

#### 1. 变量与配料桶 (Variables as Containers)
- **核心概念**：变量如同配料桶，虽然内容可变，但必须贴上标签（类型）。
- **交互挑战**：**【新手调茶师】** 根据顾客订单，准确配置出一杯奶茶。
- **关键理解**：
    - `String teaType = "Oolong";` (只能装文字)
    - `int size = 700;` (只能装整数)
    - `boolean hasIce = false;` (只有是/否两种状态)

#### 2. 基本数据类型 (Primitive Types)
- **类型匹配**：为什么不能把"珍珠"倒进存放"糖度数值"的变量里？
- **物理差异**：不同类型的原料需要不同形状的容器（内存占用）。

---

### ⚡ 第二阶段：智能计价系统 (Expressions & Math)

#### 3. 数学运算 (Mathematical Operators)
- **基础计价**：`price = base + (size * 0.05)`
- **加料计费**：`final = price + toppingCost`
- **动态计算**：实时展示收银台的计算过程。

#### 4. 复合表达式 (Compound Expressions)
- **交互挑战**：**【精算师】** 调整定价公式的参数，使得目标奶茶的售价正好命中 **25元**。
- **公式设计**：`(base * discount) + (toppings * 2)`
- **关键理解**：运算符优先级 (`*` Before `+`) 对最终价格的巨大影响。

---

### 🛡️ 第三阶段：自动质检闸门 (Conditionals & Logic)

#### 5. 布尔逻辑 (Boolean Logic)
- **真值判断**：`sugar > 0.8` (太甜？), `temp > 60` (太烫？)
- **比较运算符**：`==`, `!=`, `>`, `<`

#### 6. 逻辑门与分支 (Logic Gates & If-Else)
- **交互挑战**：**【金牌质检员】** 编写质检规则，拦截所有“致死量”（糖分过高且无冰）的奶茶。
- **代码映射**：`if (sugar > 0.8 && !hasIce) { reject(); }`
- **视觉反馈**：传送带上的机械臂根据规则自动剔除不合格产品。

---

### 📦 第四阶段：订单流批处理 (Lists & Loops)

#### 7. 列表与队列 (ArrayList)
- **数据结构**：`ArrayList<Order> queue` 就像传送带上排队的一杯杯奶茶。
- **索引访问**：`queue.get(0)` 是当前正在制作的那一杯。

#### 8. 循环遍历 (Iteration)
- **交互挑战**：**【VIP 专属通道】** 编写过滤程序，从快速移动的订单流中，利用循环自动抓取所有 VIP 订单。
- **代码映射**：`for (Order o : queue) { if (o.isVIP) { promote(o); } }`
- **算法演示**：扫描光束逐个检查，匹配成功则高亮提取。

## 🎮 演示关卡设计 (Interactive Levels)

### 关卡 1：新手调茶 (The Mixologist)

**任务**：完美复刻
**目标**：理解变量赋值与类型匹配
**交互**：
1.  左侧显示顾客需求卡片（如："大杯、红茶、无冰"）。
2.  右侧控制台提供不同类型的输入控件。
3.  学生操作控件，中间的奶茶杯实时动态生成（液面高度、颜色、冰块显示）。
4.  点击 "Serve" 提交。系统比对变量值，完全一致通过。

### 关卡 2：定价策略 (The Pricing Strategy)

**任务**：目标定价
**目标**：掌握数学表达式与运算符优先级
**交互**：
1.  系统给出一杯复杂奶茶的成本结构和目标售价（25元）。
2.  学生面对一个带有未知参数的公式：`Price = (Base + Size * [A]) * [B]`。
3.  拖动参数 A（容量系数）和 B（折扣系数）的滑块。
4.  观察实时价格曲线，直到计算结果稳定在 25.0。

### 关卡 3：质检风暴 (Quality Firewall)

**任务**：逻辑拦截
**目标**：理解复合条件判断 (AND/OR)
**交互**：
1.  传送带源源不断送来奶茶，每杯都有属性面板（糖度、温度、加料）。
2.  学生在控制台拼接逻辑积木：`IF` `[糖度 > 80%]` `AND` `[温度 > 60]`。
3.  激活闸门。符合条件（即“坏”奶茶）被机械臂推走，正常奶茶通过。
4.  漏放坏奶茶或误杀好奶茶都会扣分。

### 关卡 4：VIP 扫描器 (Stream Scanner)

**任务**：数据抓取
**目标**：掌握列表遍历与条件过滤
**交互**：
1.  底部是一条快速移动的订单流（包含普通单、VIP单、外卖单）。
2.  学生编写（选择）循环体内的逻辑：`if (order.type == "VIP")`。
3.  启动扫描器。一束激光逐个扫过订单。
4.  匹配的订单被吸入上方的 "VIP 通道"，其余继续流动。
5.  统计抓取数量，达到目标即可通关。

---

## 💡 教学亮点

1.  **所见即所得**：变量直接对应视觉元素（颜色、水位、开关）。
2.  **错误可视化**：逻辑写错了？你会看到“坏”奶茶通过闸门，或者“好”奶茶被误伤，后果直接呈现。
3.  **沉浸式角色**：不仅是写代码，更是在经营店铺，解决实际问题。
4.  **Java 代码映射**：所有操作都在右侧实时生成标准的 Java 代码，实现图形化到文本代码的认知桥梁。
