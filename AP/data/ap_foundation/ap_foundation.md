# AP-CSP 基础篇：赛博英雄 - 逻辑架构师

> **教学目标**：针对 9 年级学生，采用渐进式设计深入探讨计算逻辑的基础实现。
> **核心概念**：从基础数据类型到流程控制，再到数据结构与对象抽象。
> **视觉风格**：赛博朋克 (Cyberpunk) / 暗色黑客流。

## 📚 知识点体系架构

### 🎯 第一阶段：基础数据类型 (Building Blocks)

#### 1. 变量与内存映射 (Variables & Memory)
- **核心概念**：变量不仅是盒子，更是内存地址的别名
- **可视化**：展示 `0x4F2` 地址如何存储 `String` 引用
- **关键理解**：赋值 = 将数据写入特定内存位置

#### 2. 基本数据类型 (Primitive Types)
- **Java 强类型**：`int` (整数)、`double` (浮点数)、`String` (字符串)
- **物理差异**：不同类型在内存中的存储方式和占用空间
- **类型安全**：为什么不能将文本赋值给数字变量

#### 3. 数学运算 (Mathematical Expressions)
- **运算符优先级**：`(level * 15) * xpRate` 的计算顺序
- **复合表达式**：`currentAtk = (baseAtk * buffMultiplier) + levelBonus`
- **动态计算**：实时展示公式中每一步的计算结果

#### 4. 字符串操作 (String Manipulation)
- **动态拼接**：`String title = "LV." + level + " " + name`
- **格式化输出**：构建有意义的显示文本
- **字符串不可变**：每次拼接都会创建新的字符串对象

---

### ⚡ 第二阶段：程序流程控制 (Control Flow)

#### 5. 布尔表达式 (Boolean Expressions)
- **真值判断**：`hp < 20`、`energy > 50`、`hasShield`
- **比较运算符**：`<`、`>`、`<=`、`>=`、`==`、`!=`
- **布尔变量**：`boolean` 类型只有 `true` 和 `false` 两个值

#### 6. 条件语句 (Conditionals)
- **决策树**：`if (condition) { action } else { alternative }`
- **嵌套条件**：多层 if-else 的逻辑路径
- **电路可视化**：条件像开关，控制程序流向

#### 7. 复合逻辑 (Compound Logic)
- **逻辑 AND**：`(hp < 20 || hasShield) && energy > 50`
- **逻辑 OR**：多个条件只需满足一个
- **短路求值**：为什么顺序很重要

---

### 📦 第三阶段：数据结构与抽象 (Data Structures)

#### 8. 列表基础 (Lists - ArrayList)
- **动态数组**：`ArrayList<Item> loot = new ArrayList<>()`
- **增删操作**：`add()`、`remove()`、`get(index)`
- **动态扩容**：ArrayList 满员后如何自动扩展

#### 9. 列表遍历 (Iteration - For-Each Loop)
- **循环语法**：`for (Item i : loot) { process(i); }`
- **遍历过程**：逐个访问列表中的每个元素
- **算法演示**：在列表中寻找 `max(rarity)` 的过程

#### 10. 对象抽象 (Data Abstraction)
- **类与实例**：英雄 = `{String name, int level, double xpRate, List<String> skills}`
- **属性封装**：将相关数据组织成一个对象
- **对象内存**：一个复杂对象在底层如何被划分成不同扇区

## 🎮 演示关卡设计

### 关卡 1：核心初始化 (Core Initialization)

**知识点覆盖**：变量、基本数据类型、字符串操作

**教学目标**：
- 理解变量的本质是内存地址的别名
- 掌握不同数据类型的特点
- 学会基本的字符串拼接

**交互设计**：
- 通过输入框设置角色名称 (String)
- 滑块调节经验倍率 (double)
- 按钮增加等级 (int)

**视觉呈现**：
- 内存扇区实时显示每个变量
- 不同类型用不同颜色高亮
- 数据流注入动画，模拟赋值过程
- 右侧日志面板显示 Java 代码执行记录

**学习成果**：学生理解变量声明、赋值和类型差异

---

### 关卡 2：战力计算引擎 (Power Calculation)

**知识点覆盖**：数学运算、表达式求值

**教学目标**：
- 掌握运算符优先级
- 理解复合数学表达式
- 学会公式的逐步计算

**交互设计**：
- 调节等级和基础攻击力
- 观察公式 `(level * 15) * xpRate` 的计算过程
- 实时显示最终战力值和拼接后的称号

**视觉呈现**：
- 公式流程图，显示计算顺序
- 每个运算步骤高亮显示
- 三栏布局：输入 → 公式 → 输出
- 字符串拼接结果实时展示

**学习成果**：学生能够分解复杂公式，理解计算顺序

---

### 关卡 3：决策矩阵 (Decision Matrix)

**知识点覆盖**：布尔表达式、条件语句、复合逻辑

**教学目标**：
- 掌握布尔表达式的求值
- 理解 && 和 || 的逻辑含义
- 学会构建复合条件判断

**交互设计**：
- 滑块调节血量 (int hp)
- 开关切换护盾状态 (boolean hasShield)
- 滑块调节能量 (int energy)
- 观察复合逻辑 `(hp < 20 || hasShield) && energy > 50` 的结果

**视觉呈现**：
- 电路板式逻辑门可视化
- OR 门先计算，结果传递到 AND 门
- 电流根据真值流向不同路径
- 最终决策以大型图标展示（允许/拒绝）

**学习成果**：学生理解复杂逻辑表达式的求值过程

---

### 关卡 4：装备系统 (Inventory System)

**知识点覆盖**：列表、循环遍历、对象抽象

**教学目标**：
- 掌握 ArrayList 的基本操作
- 理解 for-each 循环的遍历机制
- 学会对象的属性访问和过滤

**交互设计**：
- 按钮筛选不同类型的装备（All / Weapon / Utility）
- 动态展示匹配条件的物品
- 每个物品显示名称、类型、稀有度

**视觉呈现**：
- 网格布局展示列表中的对象
- 过滤时动画展示遍历过程
- 不匹配的物品淡出或移除
- 匹配的物品高亮并重新排列
- 底部显示循环代码 `for(Item i : loot)`

**学习成果**：学生理解列表遍历、条件过滤和对象属性访问

## 🎯 教学流程设计

### 渐进式学习路径

1. **第一步 (关卡1)**：建立"数据在内存中存储"的心智模型
2. **第二步 (关卡2)**：理解程序如何通过计算"处理"数据
3. **第三步 (关卡3)**：掌握程序如何根据条件"做决策"
4. **第四步 (关卡4)**：综合运用数据结构和控制流程

### 跨关卡连贯性

- **统一主题**：赛博英雄角色系统贯穿始终
- **数据复用**：关卡1创建的角色数据在关卡2中使用
- **难度递增**：从单个变量 → 表达式 → 条件 → 复杂数据结构
- **概念强化**：后续关卡会复用前面学过的所有概念

---
